# AoC Client SDK Dump (Dumper-7)

Dumper-7 output for the **Ashes of Creation Alpha-2 Win64-Shipping client**
(engine version `5.6.0-438018+++game+jvs_game_rel-AOC`).

This is a static C++ representation of every `UClass`, `UStruct`,
`UFunction`, `UEnum`, and replicated property in the client binary —
exactly the information that's already loaded into the game's runtime
when you launch it. If the wire-format work in `src/` references a
property like `APlayerController::AcknowledgedPawn` or a function like
`UCharacterAppearanceComponent::OnRep_CharacterCustomization`, this is
where you can verify its offset, signature, and class membership without
opening IDA.

## What this is **NOT**

- **Not the AoC source code.** Intrepid never released source; we don't
  have it. This is a header-only reconstruction of the runtime UObject
  layout, generated by reading the in-memory class metadata that the
  client itself populates at launch.
- **Not game assets.** No textures, meshes, animations, audio, blueprints,
  level data, or pak content is included. Class names and property names
  only.
- **Not reverse-engineered logic.** The `_functions.cpp` files contain
  only function signatures and ProcessEvent stubs, not implementations.
  For implementation-level RE see `../ida-dumps/`.

## How to use this alongside the source

When source comments in `src/` reference a class, property, or function:

```cpp
// PM78: handle 15 = APlayerController::Pawn
// PM118: bHasRepLayout=1 confirmed for stably-named blocks
```

You can verify the class layout by:

1. **Search by class name** — `grep -r "class APlayerController" CppSDK/SDK/`
2. **Search by property** — `grep "AcknowledgedPawn" CppSDK/SDK/Engine_classes.hpp`
3. **Walk the JSON** — `Dumpspace/ClassesInfo.json` is structured for tools

The `IDAMappings/*.idmap` file plugs into the IDA Pro scripts in
`../ida-dumps/apply_dumper7_idmap.py` to retag every function in your
own IDA database with the SDK names — turns `sub_7FF6BD25C340` into
`UActorChannel::ReadContentBlockHeader`.

## Provenance

Generated against a copy of the retail AoC Alpha-2 client that the
contributor already had on their machine. Dumper-7 is a publicly
available open-source UE-runtime SDK extractor — no proprietary tools
were used.

The dump only contains symbol-level information that is **already
present in the shipped client binary at runtime**. Every UObject in
the game registers itself with `GUObjectArray` on startup; reading
that array is what Dumper-7 does. We're publishing the human-readable
form of public-by-construction class metadata.

## Engine version

`5.6.0-438018+++game+jvs_game_rel-AOC`

This corresponds to the AoC Alpha-2 build that was live at the time of
shutdown. Future client builds (if any) will need a fresh dump — the
class layout and property offsets WILL change between engine versions.

## Tools used

- **[Dumper-7](https://github.com/Encryqed/Dumper-7)** — Encryqed's
  UE3-UE5 SDK extractor. Runs as an injected DLL or external scanner.
- The `apply_dumper7_idmap.py` and `apply_dumper7_idmap.idc` scripts
  in `../ida-dumps/` apply the IDAMappings output to a target IDB.
